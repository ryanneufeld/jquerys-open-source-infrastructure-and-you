You're Doing It Wrong

Luckily we ran into Andrew Nacin who told us that we were doing everything wrong. Andrew called Daryl Koopersmith to help set us straight.

Other People Use WordPress Too

WordPress has an XML-RPC API which supports most of the functionality available in the admin tools. So we threw out all of our code and started over.

Now we have a sane and reliable way to manage content in WordPress.

If you want to manage a WordPress site via JavaScript, node-wordpress is the library for you!

That's Not What We Want To Do

We don't want to manage WordPress via JavaScript.

We want to just write markdown, not code. So this is helpful, but it doesn't make site deployments any easier.

Representing Code With Content

var client = wordpress.createClient({ ... });
client.newPost({
  title: "Intro to jQuery",
  content: "<p>jQuery is ...</p>"
});
becomes: 
<script>{
  "title": "Intro to jQuery"
}</script>
<p>jQuery is ...<p>

Valid HTML, invisible metadata, local preview

More Than Just One Post

Now we can represent all the data for a single post, but we still need to manage the full site.

So we use an obvious convention of organizing the HTML files into directories and use the paths as the URLs for the site. This lets us easily track what is being added, edited, and removed.

Handling Other Resources

So now we can manage all of the content, but what about other resources like images?

  • Custom XML-RPC method for arbitrary uploads.
  • Bypass standard security checks.
  • Completely free form directory of resources with explicity filenames.

grunt-wordpress

We put all of this logic into a grunt plugin so we could deploy with just one line of code.

If you want to manage a WordPress site via HTML, grunt-wordpress is the library for you!

Adding Markdown Support

The last step in this process is to get from markdown to the final HTML. To do that, we use marked and highlight-js. We also add support for YAML metadata, which we use on the jQuery Learning Center.

All that and more:

  • TOC generation from content headings
  • API site generation from XML
  • Placeholder/snippet support

If you want to manage a WordPress site via markdown, grunt-jquery-content is the library for you! (Hey! That's us!)

Managing the WordPress Install

jquery-wp-content is a custom replacement for the standard wp-content directory. Most of our customizations use the standard APIs through the existing plugin architecture.

The biggest customization is our custom installer written by Andrew Nacin, which handles auto-provisioning of new sites. Adding a new site is as simple as adding an entry to an array. The first time someone visits the site, it was auto-install and then render the page.

Theme Inheritance

WordPress' built-in theming system allows us to have parent and chlid themes. So we can have all of our common styles in a base theme and then each site can have its own theme which inherits from the base.

This is fantastic when you're managing your brand across 20 sites.
